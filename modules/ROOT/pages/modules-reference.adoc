= Modules Reference
ifndef::env-site,env-github[]
include::_attributes.adoc[]
endif::[]

Modules are similar to xref:3.4@connecting-using-transports.adoc[transports] in that they provide pluggable functionality, configured via dedicated schema, but they differ in that there is no underlying transport to send or receive data. Following is a list of the default Mule ESB modules.

[%header,cols="2*"]
|===
|Module |Description
|xref:3.4@atom-module-reference.adoc[Atom Module] |Support for consuming and creating Atom feeds.
|xref:3.4@bpm-module-reference.adoc[BPM] |Mule's support for BPM allows you to send/receive messages to/from a running process. A message from Mule can start or advance a process, the message can be stored as a process variable, and a running process can send messages to any endpoint in your Mule application.
|xref:3.4@cxf-module-reference.adoc[CXF Module] |Mule 3.0 architectural changes bring much better support for CXF, meaning it can be used just like another pipe/filter element.
|xref:3.4@using-the-mule-client.adoc[Client Module] |MuleClient and the remote dispatcher, giving simple access to the Mule server.
|xref:3.4@jaas-module-reference.adoc[JAAS Module] |Security via JAAS.
|xref:3.4@jboss-transaction-manager-reference.adoc[JBoss Transaction Manager] |JBoss transaction support.
|xref:3.4@jboss-jbpm-module-reference.adoc[jBPM Module] |http://www.jboss.org/jbpm[JBoss jBPM] is an open source BPMS and is well-integrated with Mule. One advantage of jBPM is that it is embedded directly in the Mule runtime, allowing for faster performance.
|xref:3.4@jersey-module-reference.adoc[Jersey Module] |Support for RESTful web services built using Jersey.
|xref:3.4@json-module-reference.adoc[JSON Module] |JSON data and binding support.
|xref:3.4@mule-agents.adoc[Management Module] |Mule agents for server management using JMX.
|OGNL Module |Provides a filter using http://www.ognl.org/[OGNL] expressions. For details, see xref:3.4@using-filters.adoc[Using OGNL Expressions].
|xref:3.4@pgp-security.adoc[PGP Module] |Security via PGP.
|xref:3.4@saml-module.adoc[SAML Module] _*Enterprise Edition*_ |Provides authentication and authorization capabilities to Mule based on the SAML standard. (As of Mule enterprise edition 2.2.3)
|xref:3.4@rss-module-reference.adoc[RSS Module] |Support for consuming RSS feeds
|xref:3.4@scripting-module-reference.adoc[Scripting Module] |Interface between Mule and scripting languages (currently http://groovy-lang.org/[Groovy]).
|xref:3.4@spring-extras-module-reference.adoc[Spring Extras Module] |Extensions for using the Spring framework with Mule.
|xref:3.4@sxc-module-reference.adoc[SXC Module] |A very fast streaming XPath router and filter.
|xref:3.4@xml-module-reference.adoc[XML Module] |XML based utilities (mainly filters and routers).
|===
